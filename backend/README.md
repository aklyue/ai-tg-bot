# Telegram-бот для агентов по недвижимости

Ассистент для агентов по недвижимости, который отвечает на вопросы по базе знаний о жилых комплексах, вознаграждениях, финансовых инструментах и контактах менеджеров.

## Архитектура

### Выбор RAG на векторах

Для поиска по знаниям выбрана архитектура **RAG (Retrieval-Augmented Generation) на векторных эмбеддингах** с использованием Qdrant.

#### Обоснование:

1. **Дешево**:
   - Используется локальный Qdrant (бесплатно)
   - Эмбеддинги: `text-embedding-3-small` (дешевая модель OpenAI)
   - LLM: `gpt-4o-mini` (оптимальное соотношение цена/качество)
   - Стоимость одного запроса: ~$0.001-0.003

2. **Быстро**:
   - Векторный поиск: 10-50ms на запрос
   - Эмбеддинги: кэшируются
   - Общее время ответа: 1-2 секунды (вместо 30+ секунд у графовых систем)

3. **Простота поддержки**:
   - Минимум зависимостей
   - Легко добавлять новые источники знаний
   - Прозрачная логика работы

#### Альтернативы, которые НЕ выбраны:

- **Граф знаний** (Neo4j): сложнее в поддержке, дороже, медленнее на простых запросах
- **Гибридный подход**: избыточен для текущих задач, добавляет сложность без значительного выигрыша

### Стек технологий

- **Бот**: aiogram 3.x
- **RAG**: LangChain + Qdrant + OpenAI embeddings
- **STT**: Soniox API
- **LLM**: Bothub.ru (OpenAI-совместимый API)
- **Парсинг**: Playwright
- **Хранение**: Локальный Qdrant (в продакшене заменить на облачный)

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <repo-url>
cd test-task-bot
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните `.env`:
- `TG_BOT_TOKEN` - токен Telegram бота от @BotFather
- `SONIOX_API_KEY` - ключ Soniox для STT
- `OPENAI_API_KEY` - ключ Bothub.ru
- `OPENAI_BASE_URL` - `https://bothub.chat/api/v2/openai/v1`

### 4. Инициализация базы знаний

Перед первым запуском нужно заполнить базу знаний:

```bash
python parser.py
```

Или для обновления базы знаний (повторяемый парсинг):

```bash
python parser.py
```

### 5. Запуск бота

```bash
python -m bot.main
```

Или через Docker:

```bash
docker-compose up --build
```

## Использование

### Бот

1. Напишите боту в Telegram
2. Отправьте текстовое сообщение или голосовое
3. Бот ответит на основе базы знаний

### Парсинг

Для обновления базы знаний:

```bash
python parser.py
```

Парсер автоматически:
- Парсит все сайты из `KNOWLEDGE_SOURCES["websites"]`
- Скачивает и индексирует документы из `KNOWLEDGE_SOURCES["documents"]`
- Логирует статус каждого источника
- Обрабатывает дубликаты (документы добавляются, не перезаписываются)

## Устранение неполадок

### Ошибка подключения к Telegram API

Если вы видите ошибку:
```
ERROR:root:Telegram network error: HTTP Client says - ClientConnectorError: Cannot connect to host api.telegram.org:443 ssl:default
```

Это означает, что бот не может подключиться к серверам Telegram. Это часто происходит в регионах, где Telegram заблокирован (например, в России).

**Решение: Настройте прокси**

1. Установите VPN или прокси-сервер (например, v2ray, clash, или любой SOCKS5/HTTP прокси)
2. Откройте файл `.env`
3. Установите переменную `TELEGRAM_PROXY`:

```bash
# Для HTTP прокси (например, от v2ray/clash):
TELEGRAM_PROXY=http://127.0.0.1:7890

# Для SOCKS5 прокси:
TELEGRAM_PROXY=socks5://127.0.0.1:1080

# Для другого HTTP прокси:
TELEGRAM_PROXY=http://proxy-server:8080
```

4. Перезапустите бота

**Проверка работы прокси:**
```bash
# Проверьте, работает ли прокси:
curl -x http://127.0.0.1:7890 https://api.telegram.org
```

Если команда возвращает ответ (не ошибку подключения), прокси работает корректно.

## Разработка

### Добавление нового источника знаний

В `parser.py` добавьте URL в соответствующий список:

```python
KNOWLEDGE_SOURCES = {
    "websites": [
        "https://example.com",
    ],
    "documents": [
        "https://example.com/document.pdf",
    ]
}
```

Затем запустите парсинг:

```bash
python parser.py
```

### Изменение промпта

Промпт для LLM находится в `bot/rag.py` в функции `get_answer()`.

### Логирование диалогов

Диалоги логируются в память (`dialogue_history` в `bot/main.py`). В продакшене заменить на БД (PostgreSQL/MongoDB).
